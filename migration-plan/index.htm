3            OWC Plugin System (Metadata processor)

 

#

Description

Note

OWC-26

Plugin system

Also known as Metadata processor. The user defines a plugin composed of an input web-component that produces an input model, a sequence of web-workers to elaborate the input model and a visualizer component to show the elaborated model. The plugin system is an orchestrator of the plugins defined by the user. In other words Metadata processor is a plugin system client-side for scripts written in JavaScript language.

 

 

OWC-27

Sentinel-1 interferometry plugin

An example of operative implementation of Plugin system (OWC-8).

 

OWC-28

Plugin system documentation

The detailed documentation for developers and users of OWC plugin system.

 

 

OWC-29

Image processing client side investigation

A simple proof of concept to explore the possibility to process small images client-side (possible evolution of plugin system).

 

 

OWC-30

Plugin system graphical editor

Development of an online graphical editor to create and integrate new plugins to run on OWC.

Exportable local configuration

The local configuration of OWC app are stored in the browser using the local storage. Must be possible to export and import this configurations to share or move the configurations between different browser instances.

Evolution of the current list or map to host the model from different data-sources, e.g. to show the cart products and  the search results together in the same map.

The product classes hierarchy API is missing. This API is very useful to improve semantic schema definition in the DHuS Addons, defining metadata indexes mapping for each product classe.

7            Dynamic layout and meta-component

 

#

Description

Note

OWC-46

Dynamic layout

In the OWC 1.0 version the layout of components is static. Provide an engine to manage customizable grid-layouts represented as a javascript objects, with a grid definition and the components to load inside the cells.

 

 

OWC-47

Meta-component (basic customization for component: e.g. layout, transparency, background)

Based on Dynamic layout, the meta-component will be the parent class for every component in OWC application, where is possible to define transparency, background, layout via graphical user interface.

 

 

OWC-48

Dynamic layout editor

Graphical editor to define or update a dynamic layout of a component.

 

 

OWC-49

Web component Graphical Creator

Graphical Components designer, inspired from https://polymer-designer.appspot.com/ to create a new component defining a dynamic layout via graphical user interface and putting inside the layout components exploiting drag and drop. The user could download the code of the new component and integrate it in the application.
